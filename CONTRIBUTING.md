# Contributing

感谢你为本项目做贡献！

## 提交问题前
- 先搜索是否已有相同 Issue。
- 提供复现步骤、预期结果、实际结果。
- 如涉及界面问题，请附截图或录屏。

## 提交 Pull Request
- 每次 PR 聚焦一个主题，避免混入无关改动。
- 代码需保持清晰，命名语义化。
- 新功能请补充必要文档；涉及行为变更请更新 README。
- 提交前请确保本地可运行、关键流程可验证。

## Commit 规范
- 推荐使用 `type(scope): subject` 格式。
- 常用 type：`feat`、`fix`、`docs`、`refactor`、`build`、`chore`。

## 代码风格
- 避免重复逻辑，公共能力请抽取复用。
- 变更应尽量小而明确，便于评审和回滚。

## 行为准则
请先阅读并遵守 `CODE_OF_CONDUCT.md`。
